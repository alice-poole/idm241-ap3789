IDM241 Beta Description

Last week, I made hovering over a Pinterest post more fun with a fluid bump animation to the buttons.
Since then, I've done some thinking. I could expand that microinteraction system, but at the end of the day, Pinterest (probably) won't use it.
But I know someone who would really use a website with fluid bump animations: me! So I recycled the concept for an all-new button system.
First, let's look at default HTML buttons.


Unstyled Buttons

You've seen them, you've used them, but how bad are they really?

What To Do About It

Goals

  1. Build on smooth sliding motion developed for my last demo.
  2. Create a fun way to interact with buttons on a stylish portfolio site.
  3. Explore current design trends of tactile 3D feel.
  4. Why not let buttons...be buttons?

Hover Off

Trigger

  • Hover off a button.

Rules

  • Cursor changes to default arrow.
  • Over 200ms, the white button top will move up by .25rem.

Feedback

  • Cursor changes to default arrow.
  • Over 200ms, the white button top will move up by .25rem.

Modes


Click: Springy Buttons

This variant feels right for buttons that are pressed and spring back, like a spacebar.
This will be used for buttons that open modals or turn pages.

Trigger

  • Click on a springy button.

Rules

  • Over 200ms, the white button top will move down by .25rem, fully obscuring the gray button side.
  • Then, over another 200ms, the white button top will move up by .25rem, returning to the hover position.

Feedback

  • Over 200ms, the white button top will move down by .25rem, fully obscuring the gray button side.
  • Then, over another 200ms, the white button top will move up by .25rem, returning to the hover position.

Loops


Modes


Click: Sticky Buttons

This variant feels right for buttons that are pressed and stick, like toggles in a switchboard.
This will be used for things like page selection in a navbar.

Trigger

  • Click on a sticky button.

Rules

  • Over 200ms, the white button top will move down by .25rem, fully obscuring the gray button side.
  • Over 200ms, excluding the newly selected button, all sticky buttons' white button tops will move to .5rem above their gray button side. (Any previously selected sticky button resets to the default unpressed position.)

Feedback

  • Over 200ms, the white button top will move down by .25rem, fully obscuring the gray button side.
  • Over 200ms, excluding the newly selected button, all sticky buttons' white button tops will move to .5rem above their gray button side. (Any previously selected sticky button resets to the default unpressed position.)
