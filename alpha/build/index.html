IDM241 Alpha Description

I hate Pinterest's post microinteractions. Last time, I talked about why I don't like them and my plan to improve them. (See part 1 here!)
Now, I improved them! Here is a working version of my new design. (Look how far she's come!)


Current Design


Hover On

Trigger

  • Hover on a post.

Rules

  • Cursor changes to magnifying glass.
  • Save button appears, slides down from top by 21px then back 9px over .25 seconds.
  • After .75 seconds, save button wiggles back and forth by 5 degrees on a 1 second loop.
  • Original post link appears, slides up from bottom by 21px then back 9px over .25 seconds.
  • Share button appears, slides up from bottom by 21px then back 9px over .25 seconds.
  • More options appears, slides up from bottom by 21px then back 9px over .25 seconds.

Loops

Modes

  • More options button appears.

Hover Off

Trigger

  • Hover off a post.

Rules

  • Cursor reverts to arrow.
  • Save button disappears.
  • Original post link disappears.
  • Share button disappears.
  • More options button disappears.

Feedback

  • Cursor reverts to arrow.
  • Save button disappears.
  • Original post link disappears.
  • Share button disappears.
  • More options button disappears.

Loops


Modes

  • Save button disappears.
  • Original post link disappears.
  • Share button disappears.
  • More options button disappears.

Lessons Learned

  • Complex hover off animations are more trouble than they're worth. Focus on rewarding users for engagement, not disengagement.
  • CodePen is a great tool for coding in a vacuum. I can't promise I won't break everything, but I CAN break a different, smaller everything!
  • Busy wiggle animations are great to draw users' attention and curiosity!
  • Buttons clutter the visually-important posts. To keep things clear, distracting animations can pressure users to engage or make them go away by exiting. Posts clear!
